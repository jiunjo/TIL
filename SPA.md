## 📘 오늘 배운 내용 요약

SPA(Single Page Application)는 **최초 한 번만 HTML/JS/CSS를 받아온 뒤**,  
페이지 이동 시마다 전체 페이지를 다시 불러오는 대신  
**필요한 데이터만 비동기 요청으로 받아와 화면의 일부만 업데이트하는 방식**이다.

즉, 전통적인 MPA처럼 매번 새 문서를 요청하는 것이 아니라,  
브라우저에 이미 올라와 있는 JavaScript가 **라우팅 + 렌더링을 담당**하고  
사용자에게 보여지는 화면을 클라이언트에서 즉시 변경한다.

---

## 🔍 핵심 개념 정리

### 1. 기존 MPA(Multi Page Application)
- 각 URL마다 별도의 HTML 파일을 가진다.
- `/`, `/about`, `/diary`로 이동할 때마다 **서버에서 새로운 HTML을 통째로 로딩**.
- 전체 새로고침이 발생해 화면 깜빡임이 있다.

### 2. SPA 방식
- 최초 진입 시 `index.html` 한 번만 로딩 → 이후는 JS 라우터가 화면 제어.
- URL이 바뀌어도 실제 페이지 이동이 일어나지 않고,  
  JavaScript가 해당 경로에 맞는 컴포넌트를 렌더링한다.
- 서버에서는 주로 **JSON API 응답**만 받아오면 된다.

### 3. 비동기 통신
- `fetch`, `axios`로 서버 요청을 보낼 때 **비동기** 처리로 진행.
- 응답을 기다리는 동안에도 UI는 멈추지 않고 동작.
- 만약 동기 방식이었다면 응답 오기 전까지 전체 UI가 **멈추는 문제** 발생.

### 4. 부분 렌더링
- MPA: 새 HTML 전체를 받아와 문서 전체 교체.
- SPA: React의 가상 DOM이 바뀐 부분만 실제 DOM에 반영.
- 그 결과 헤더/사이드바 같은 공통 레이아웃은 유지되고,  
  중간 콘텐츠만 교체되어 **부드러운 전환**이 가능하다.

### 5. 장점
- 앱처럼 부드러운 전환(UX)
- 공통 레이아웃을 유지하면서 내용만 변경 가능
- 빠른 인터랙션 처리

### 6. 단점
- 초기 로딩 시 JS 번들이 크면 화면 늦게 뜬다.
- 기본적으로 SEO가 불리함  
  (검색엔진이 JS 실행 후 렌더링까지 기다려야 함)

---

## 🔎 SEO 문제는 실제로 치명적인가?

SPA는 기본 구조상 SEO가 불리하지만,  
**현재는 완전히 해결할 방법이 여러 개 존재한다.**

### SEO 해결 방법
1. **SSR(Server Side Rendering)**  
   - Next.js 방식  
   - 서버에서 미리 HTML을 만들어 보내므로 SEO 매우 유리.

2. **SSG(Static Site Generation)**  
   - 미리 HTML을 생성해 배포 (Next.js SSG, Gatsby, Astro 등)

3. **Pre-rendering (프리렌더링)**  
   - SPA여도 prerender.io 같은 서비스로  
     검색엔진용 HTML 미리 생성 가능.

> 즉, “SPA = SEO 치명적 문제”는 예전 얘기이며  
> 지금은 충분히 해결 가능하다.

## 🔥 SEO가 필요 없는 사이트 = React만 써도 충분한 경우

다음과 같은 서비스는 **SEO 의미가 거의 없기 때문에**  
굳이 Next.js로 SSR/SSG를 쓰지 않고 **React만으로 구성해도 전혀 문제 없음.**

### 1) 로그인 기반 서비스 (Auth 필수)
- 관리자 페이지(Admin)
- 사내 인트라넷
- 개인 설정 대시보드
- 마이페이지 기반 웹앱

→ 검색엔진이 로그인할 수 없기 때문에 SEO 자체가 의미 없음.

### 2) 앱형 UI 중심 서비스
- To-Do 앱
- 메신저/채팅 서비스
- 음악/영상 플레이어
- 캘린더/스케줄러

→ 사용자의 상호작용이 핵심이며 SEO 필요 없음.

### 3) 내부/관리자 툴
- 쇼핑몰 관리자 화면
- 분석/모니터링 대시보드
- 사내 리포트 툴

→ 외부 유입 필요 없음 = SEO 무관.

### 4) 로그인이 필요한 커뮤니티 내부 페이지
- 피드 / 타임라인
- DM/메시지
- 알림 페이지

→ 검색엔진이 접근할 수 없음.

### 5) 단순한 포트폴리오 편집 UI
- 팬시한 UI만 필요한 경우  
- 외부 검색 유입 받을 필요 없을 때

## 🧩 한 줄 요약

> 앱처럼 부드럽게 동작하는 웹 구조다.  
> SEO는 기본적으로 불리하지만 SSR/SSG/프리렌더링으로 해결되고,  
> SEO 필요 없는 서비스는 React만으로 충분하다.**
