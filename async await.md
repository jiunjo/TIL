## 📘 오늘 배운 내용 요약
async/await는 **비동기(Promise) 기반 작업을 동기 코드처럼 읽기 쉽게 만드는 문법**이다.  
await는 Promise의 완료를 **기다렸다가(next line)** 실행하게 만들지만,  
브라우저 전체(UI, 렌더링, 애니메이션)는 **절대 멈추지 않는다.**

---

## 🔍 핵심 개념 정리

### ✔ 1. async
- 함수 앞에 붙이며, 이 함수는 **Promise를 반환하는 비동기 함수**가 된다.
- async 함수 내부에서만 **await**를 사용할 수 있다.

### ✔ 2. await
- Promise가 **완료될 때까지 다음 줄 실행을 잠시 멈춘다.**
- 하지만 **UI는 멈추지 않는 논블로킹 방식**이다.
- 즉 “코드는 동기처럼 보이지만 실제로는 비동기적으로 동작”한다.

### ✔ 3. 왜 필요한가?
- 비동기 흐름을 then() 체인 없이 **순서대로 읽기 쉽게** 만들어준다.
- 서버 요청(fetch/axios) 후 데이터로 UI 업데이트하는 흐름을 명확하게 관리한다.
- try/catch로 **비동기 에러 처리**가 쉬워진다.

### ✔ 4. 동기 vs 비동기 비교
- 동기(블로킹): 오래 걸리는 작업이 있으면 **전체 UI가 멈춤**.
- 비동기(async/await): 오래 걸려도 **UI는 정상 동작**, 코드 흐름만 대기한다.

### ✔ 5. await가 필요한 이유
- await를 쓰지 않으면 fetch가 완료되기 전에 아래 코드가 실행되어  
  **데이터가 없는 상태에서 UI 업데이트 → 오류 가능성**이 커진다.
- await는 “데이터 받아온 뒤 다음 작업 수행”을 확실히 보장한다.

전체 UI는 멈추지 않는 ‘비동기 + 동기 스타일 제어’ 문법이다.**
